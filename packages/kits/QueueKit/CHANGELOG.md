# QueueKit Changelog

## 2026-06-29 тАФ CAND-023: Filesystem identifier normalization (planned security hardening)

**Branch:** `secfix/c-queuepath`

**Security classification:** MEDIUM тАФ path traversal via caller-supplied identifiers.

`FilesystemBackend` (Swift, Rust, Python) built on-disk paths тАФ job filenames,
stream-embedded components, and signal file names тАФ from caller-supplied
identifiers (`stream_id`, job id) WITHOUT validating them to safe single path
components. A `stream_id` or job id containing `/`, `\`, or `..` could escape
the queue root or collide with paths outside the maildir. An absolute path
passed as a `stream_id` (e.g. `/etc/passwd`) would similarly escape.

**Fix:** Every identifier used to construct a filesystem path is validated before
any OS call. The check rejects:
- empty strings
- `.` or `..` (dot directory components)
- `/` or `\` (POSIX and Windows path separators)
- ASCII control characters (0x00тАУ0x1F, 0x7F)

Legitimate identifiers тАФ 32-char hex job ids generated by `JobID.generate()`,
kebab-case stream names like `"encode"` or `"encode-corpus"` тАФ all pass.
Validation throws `QueueError.invalidIdentifier` (Swift), returns
`Err(QueueError::InvalidIdentifier)` (Rust), or raises `InvalidIdentifier`
(Python) before any filesystem path is constructed.

**Scope (all three ports):**
- `FilesystemBackend.write()` / `FilesystemBackend::write()` / `write()` тАФ validates `stream_id` + job id before filename construction
- `FilesystemBackend.writeBatch()` / `write_batch()` тАФ same validation per-job
- `FilesystemBackend.complete()` / `complete()` тАФ validates job id before signal file path construction
- `FilesystemBackend.completeBatch()` / `complete_batch()` тАФ validates all job ids before any filesystem use
- `QueueError.invalidIdentifier` (Swift) тАФ new error case
- `QueueError::InvalidIdentifier` (Rust) тАФ new error variant
- `InvalidIdentifier(QueueError)` (Python) тАФ new error class

**Tests added (9 per port, 27 total):**
- `rejectsDotDotStreamID` тАФ `..` as stream_id is rejected at write
- `rejectsDotDotJobID` тАФ `..` as job id is rejected at write
- `rejectsForwardSlashInStreamID` тАФ `/` in stream_id is rejected
- `rejectsBackslashInJobID` тАФ `\` in job id is rejected
- `rejectsAbsolutePathAsStreamID` тАФ absolute path as stream_id is rejected
- `rejectsControlCharacterInJobID` тАФ control character in job id is rejected
- `acceptsLegitimateIdentifiers` тАФ hex job id + kebab stream name pass through cleanly
- `rejectsDotDotJobIDOnComplete` тАФ `..` job id rejected at complete
- `rejectsForwardSlashJobIDOnComplete` тАФ `/` job id rejected at complete
